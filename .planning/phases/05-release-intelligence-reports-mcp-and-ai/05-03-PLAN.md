## Phase Goal

**As a** squad planner, **I want to** run a local MCP server and chat with an AI assistant that answers using live planning data, **so that** I can ask questions about release health and apply controlled changes through the assistant.

## Artifacts this phase produces

- `src/mcp/server.ts` — Local MCP HTTP server binding to `mcpHost:mcpPort` from settings
- `src/mcp/tools/index.ts` — Tool registry with metadata, validation, and execution routing
- `src/mcp/tools/read.ts` — Read tools: releases, active release, sprints, capacity, features, stories, backlog, timeline, capacity report, leakage report
- `src/mcp/tools/suggest.ts` — Suggestion tools: scope adjustments, story redistribution, capacity risks, late features, release status explanation
- `src/mcp/tools/write.ts` — Write tools: create/update feature, create/update story, move story between sprints, update story status
- `src/mcp/tools/safe.ts` — Safe wrappers: close sprint, reopen sprint, cancel story, cancel feature (all require `confirmed: true`)
- `src/lib/ai.ts` — OpenAI-compatible client with `streamChatWithTools(messages, tools)`
- `src/app/api/chat/route.ts` — `POST` chat endpoint that streams AI responses and executes tool calls
- `src/app/assistant/page.tsx` — Operational assistant page replacing placeholder
- `src/features/assistant/assistant-chat.tsx` — Chat UI with user/assistant message bubbles, streaming indicator, and tool-call rendering
- `src/features/assistant/quick-prompts.tsx` — Predefined prompt buttons: "Is my release on track?", "Which sprints are over capacity?", "What are my risky features?", "Suggest story redistribution", "Summarize leaked stories", "Which stories lack estimates?"
- `src/features/assistant/confirm-dialog.tsx` — Modal for dangerous operation confirmation
- `tests/mcp.test.ts` — Unit tests for MCP tool registration, read tools, write tools, and safe-gate behavior
- `tests/e2e/assistant.spec.ts` — E2E tests for chat rendering, quick prompts, and confirmation flow

## Objective

Deliver the local MCP server and AI assistant. The MCP server exposes read, suggestion, write, and safe tools over HTTP on localhost. The AI assistant chat uses an OpenAI-compatible API with tool definitions, streams responses, and requires explicit user confirmation before executing sensitive operations.

Purpose: Enable AI-powered planning assistance grounded in real local data.
Output: Working MCP server, tool registry, chat API, assistant UI, quick prompts, and confirmation flow.

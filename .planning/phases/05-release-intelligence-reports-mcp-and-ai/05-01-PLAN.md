  <action>
    Create `src/lib/progress.ts` with two exported functions:
    1. `calculateReleaseProgress(releaseId)` — Query all non-canceled stories for the release. Sum total story points and finished story points. If total story points > 0, return `finishedSP / totalSP` as a number 0–1. Otherwise return `finishedCount / totalCount` (fallback to story count). If totalCount is 0, return 0.
    2. `calculateSprintProgress(sprintId)` — Query all non-canceled stories assigned to the sprint. Sum `estimatedDays` for total planned and for stories with status `DONE`. Return `finishedDays / plannedDays`; if plannedDays is 0, return 0.

    Create `src/lib/dashboard.ts` with `getDashboardData(releaseId)` that returns:
    - `release`: basic release info
    - `progress`: release progress percentage (0–100)
    - `totalCapacityDays`: sum of `netCapacityDays` across all sprints in the release
    - `plannedEffortDays`: sum of `estimatedDays` for all non-canceled stories assigned to release sprints
    - `remainingCapacityDays`: `totalCapacityDays - plannedEffortDays` (can be negative)
    - `risk`: "Over capacity" if `plannedEffortDays > totalCapacityDays`, otherwise "On track"
    - `featureCount`: count of non-canceled features in the release
    - `storyCount`: count of non-canceled stories in release features
    - `finishedStoryCount`: count of non-canceled stories with status `DONE`
    - `leakedStoryCount`: count of distinct stories with `LeakageHistory` records where `originSprint.releaseId === releaseId`
    - `sprints`: array of sprint rows (see below)

    Each sprint row must include: sprint id, name, goal, status, period, grossCapacityDays, netCapacityDays, plannedEffortDays, remainingCapacityDays, occupancyPercentage, progressPercentage, and overCapacity boolean.

    Create `src/lib/alerts.ts` with `detectAlerts(releaseId)` that returns an array of typed alert objects `{type, severity, message, link?}` for these conditions:
    - Over-capacity sprint: `plannedEffortDays > netCapacityDays` → severity `warning`
    - Over-capacity release: total planned > total net capacity → severity `danger`
    - Feature without stories: any non-canceled feature has zero non-canceled stories → severity `info`
    - Story without estimates: any non-canceled story has `estimatedDays == null || estimatedDays === 0` → severity `info`
    - Leaked stories: `leakedStoryCount > 0` → severity `warning`
    - Sprint without goal: any sprint has empty/null goal → severity `info`
    - Empty sprint: any sprint with `IN_PROGRESS` or `PLANNED` status has zero assigned non-canceled stories → severity `info`

    Write `tests/dashboard.test.ts` covering progress calculation (story-point path, fallback path, zero-total path), alert detection for each condition, and dashboard aggregation with mocked Prisma queries.
  </action>
  <acceptance_criteria>

    - `calculateReleaseProgress` returns 0.25 when 1 of 4 stories is done and no story points exist.
    - `calculateSprintProgress` returns 0.6 when 3 of 5 planned days are finished.
    - `detectAlerts` returns an over-capacity alert when a sprint's planned effort exceeds net capacity.
    - `detectAlerts` returns a leaked-stories alert when `LeakageHistory` exists for the release.
    - `getDashboardData` includes `totalCapacityDays`, `plannedEffortDays`, and `risk` fields.

## Phase Goal

**As a** squad planner, **I want to** generate planning and tracking reports and export them to CSV or Excel, **so that** I can share release status outside the app and keep offline records.

## Artifacts this phase produces

- `src/lib/reports.ts` — `generateReport(type, releaseId)` for all 7 report types
- `src/lib/export.ts` — `csvFromRows(rows)`, `excelFromSheets(sheets)` export utilities
- `src/app/reports/page.tsx` — Operational reports page replacing the Phase 1 placeholder
- `src/features/reports/report-list.tsx` — Report type selector, release selector, generate button
- `src/features/reports/report-viewer.tsx` — Report table display with CSV and Excel export buttons
- `src/app/api/reports/route.ts` — `POST` to generate a report by type and releaseId
- `src/app/api/reports/export/route.ts` — `POST` to download CSV or Excel from report data
- `tests/reports.test.ts` — Unit tests for report generation and export utilities
- `tests/e2e/reports.spec.ts` — E2E tests for report generation, display, and download

## Objective

Deliver the reports screen with seven report types, tabular display, and CSV/Excel export. Replace the reports placeholder with an operational UI where the user selects a report type and release, generates the report, views it in a table, and downloads it.

Purpose: Provide offline-accessible planning data exports and structured report views.
Output: Working reports page, report generation library, export utilities, and tests.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> local release API | Release status changes arrive from the UI and must be validated before touching persistence. |
| application -> SQLite file | Baseline rows are written locally and must remain consistent with the release transition transaction. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01-01 | Tampering | `src/lib/releases.ts` | mitigate | Validate transition order server-side and reject direct skips before any database write. |
| T-09-01-02 | Tampering | baseline snapshot writes | mitigate | Create snapshot rows inside the same transaction as the status change and guard against duplicate capture. |
| T-09-01-03 | Information Disclosure | local database | accept | No new remote exposure is added; the data remains localhost-only in SQLite. |
</threat_model>

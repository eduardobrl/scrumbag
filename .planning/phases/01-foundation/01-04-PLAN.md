<objective>
Implement automatic Excel file ingestion from a watched folder: SHA-256 content hashing prevents duplicate imports, chokidar detects file changes, SheetJS parses `.xlsx` data, and the UI allows configuring the sync folder and viewing sync status.

Purpose: SYNC-01 and SYNC-02 require the app to detect Excel file changes and import data reliably. This plan delivers the complete sync vertical slice: file watcher → parser → importer → UI feedback.
Output: Users place `.xlsx` files in a configured folder and see imported backlog items appear automatically, with deduplication via content hashing.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system ↔ Bun process | Untrusted Excel files from OneDrive could be malicious or malformed |
| Browser ↔ Bun server | Sync folder path is user-configurable; must not allow path traversal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-10 | Tampering | Path traversal via sync folder path | mitigate | server.ts validates folderPath is a relative path within project directory; rejects absolute paths and ".." segments |
| T-01-11 | Tampering | Malicious Excel file (macros, exploits) | mitigate | SheetJS parses only worksheet data; no macro execution. File size limit (50MB) enforced before parsing. |
| T-01-12 | Denial of Service | Very large Excel file | mitigate | Reject files > 50MB before parsing. Timeout import after 30 seconds. |
| T-01-13 | Tampering | SQLite injection via file_hashes table | mitigate | Upsert uses parameterized query: INSERT OR REPLACE INTO file_hashes (path, hash, synced_at) VALUES (?, ?, CURRENT_TIMESTAMP) |
| T-01-14 | Information Disclosure | Error messages reveal file system paths | mitigate | API error responses return generic messages; log full details server-side only |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | All packages Approved in RESEARCH.md audit; no [ASSUMED]/[SUS] packages |
</threat_model>

    1) Create releases with id TEXT PRIMARY KEY, name TEXT NOT NULL, description TEXT, start_date TEXT NOT NULL, end_date TEXT NOT NULL, status TEXT NOT NULL DEFAULT 'planned' CHECK(status IN ('planned','active','closed')), created_at TEXT DEFAULT CURRENT_TIMESTAMP, updated_at TEXT DEFAULT CURRENT_TIMESTAMP.
    2) Create release_features with id TEXT PRIMARY KEY, release_id TEXT NOT NULL REFERENCES releases(id) ON DELETE CASCADE, feature_id TEXT NOT NULL REFERENCES backlog_items(id) ON DELETE CASCADE, start_sprint_id TEXT REFERENCES sprints(id) ON DELETE SET NULL, end_sprint_id TEXT REFERENCES sprints(id) ON DELETE SET NULL, board_order INTEGER NOT NULL DEFAULT 0, added_at TEXT DEFAULT CURRENT_TIMESTAMP, UNIQUE(release_id, feature_id).
    3) Add sprints.release_id as a nullable TEXT column when missing. Use addColumnIfMissing.
    4) Create one default release named "Release inicial" if any sprint has release_id IS NULL. Its dates should span MIN(start_date) to MAX(end_date) for those sprints. Then update existing null-release sprints to that release id. Do not delete or rewrite existing sprint items.
    5) Add indexes for sprints.release_id, release_features.release_id, release_features.feature_id, and release_features start/end sprint ids.
    6) Add SQLite triggers that abort INSERT and UPDATE on backlog_items when type IN ('story','bug') and parent_id is null or parent is not a feature. Add a trigger that aborts deleting a feature while story/bug children still exist. Use CREATE TRIGGER IF NOT EXISTS.
  </action>

    - Existing local databases with sprints migrate without losing sprint_items.
    - Existing sprints with null release_id are assigned to "Release inicial".
    - New sprints can store release_id.
    - SQLite rejects story/bug rows without a feature parent.
    - SQLite rejects deleting a feature that still has story/bug children.
    - No new runtime dependency is introduced.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to local Bun API | Release, sprint, and backlog hierarchy mutations cross into SQLite persistence |
| Repository to SQLite | Database constraints must preserve planning invariants even when code paths change |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Story/bug created without feature parent | mitigate | Zod schema, BacklogRepository validation, and SQLite triggers |
| T-04-02 | Tampering | Sprint created outside a release | mitigate | Release-scoped sprint endpoint and repository/API validation |
| T-04-03 | Tampering | Story from another release enters sprint scope | mitigate | SprintRepository validates parent feature release membership before addItem |
| T-04-04 | Denial of Service | Unsafe schema migration on existing local DB | mitigate | Idempotent additive migration, default release backfill, no destructive table rebuild |
| T-04-05 | Repudiation | Feature added during active release without notice | accept | Store added_at and expose warning in later UI; full audit log is out of scope |
</threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> local story API | Users can submit story estimate edits and optional reasons; server-side rules decide what becomes audit history. |
| application -> SQLite file | Story changes and audit rows must stay transactionally consistent in local persistence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Tampering | `src/lib/stories.ts` | mitigate | Compare persisted old values to validated new values server-side and write audit rows inside the story update transaction. |
| T-10-01-02 | Repudiation | estimate audit trail | mitigate | Store timestamp, field, old value, new value, and optional reason for every post-go-live estimate change. |
| T-10-01-03 | Information Disclosure | story history UI | accept | The app remains localhost-only and displays only local planning data already visible to the operator. |
</threat_model>

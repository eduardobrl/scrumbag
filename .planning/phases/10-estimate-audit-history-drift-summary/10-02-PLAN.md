<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| release page -> local data layer | The release detail page reads aggregate drift from local persistence and must not mutate the baseline. |
| application -> SQLite file | Drift totals derive from immutable baseline rows plus current story rows in the local database. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Tampering | drift calculation | mitigate | Treat baseline rows as read-only and compute drift on demand from baseline item story IDs. |
| T-10-02-02 | Information Disclosure | release detail UI | accept | Drift exposes local release planning values already visible elsewhere in the app. |
| T-10-02-03 | Denial of Service | aggregate query | mitigate | Query only the current release baseline/story set and keep calculations in memory for expected v1 scale. |
</threat_model>

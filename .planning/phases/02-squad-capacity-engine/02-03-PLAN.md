<objective>
Add waste/overhead accounting and manual override to the capacity engine. Users can configure a global waste percentage (meetings, support, incidents), see waste deducted transparently in the capacity breakdown, and manually override capacity for specific members and dates.

Purpose: Realistic capacity planning must account for non-discretionary time. The override allows corrections for edge cases the algorithm cannot predict (e.g., partial availability, onboarding overhead).
Output: Updated capacity service with waste and override, new config UI, and enhanced capacity view.
</objective>

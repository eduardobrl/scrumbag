# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] **CONSTRAINT: Port 3000 TIME_WAIT during verification** — When testing the Bun server, previous processes leave port 3000 in TIME_WAIT state, causing `EADDRINUSE` errors. The sed-based port-switching approach also created duplicate port lines. Structural mitigation: use a dedicated test port (e.g., 3002) for automated verification, or wait 60s between server restarts. Always verify `netstat` before assuming port is free.
- [ ] **CONSTRAINT: Bun.crypto.subtle undefined on Windows** — The existing sync watcher crashes with `TypeError: undefined is not an object (evaluating 'Bun.crypto.subtle')` on Bun v1.3.13 Windows x64. This is a pre-existing issue in `src/sync/hash.ts` unrelated to Phase 2 work. Structural mitigation: do not let sync errors block API endpoint verification; use a test port and focus on `/api/squad` and `/api/absences` responses.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Port-switching with sed | Changing server port via sed for testing risks leaving duplicate port lines if interrupted | advisory | Use a dedicated test script or environment variable for test ports instead of mutating server.ts |
| TIME_WAIT blindness | Not checking `netstat` before starting a test server wastes time on false EADDRINUSE errors | advisory | Always run `netstat -ano \| grep 3000` before `bun run dev` in test loops |
| Sync error distraction | Existing `Bun.crypto.subtle` crash in sync watcher distracts from validating new API endpoints | advisory | Isolate API verification from full server startup by testing endpoints directly on a clean port |

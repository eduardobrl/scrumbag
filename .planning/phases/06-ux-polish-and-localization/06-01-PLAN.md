## Phase Goal

**As a** squad planner working in Brazil, **I want to** switch release context from the header, see calendar sprint ranges with business day counts, click larger action buttons, and use the app fully in Portuguese, **so that** daily planning is faster, more comfortable, and matches my working language.

## Artifacts this phase produces

- `src/components/ui/icon-button.tsx` — Shared icon-only button component with 40x40px hit area and tooltip
- `src/components/release-switcher.tsx` — Client component dropdown in header for release context switching
- `src/messages/pt-BR.json` — Brazilian Portuguese translation keys for all UI strings
- `src/messages/en.json` — English fallback translation keys
- `src/i18n/request.ts` — next-intl request configuration
- `src/i18n/navigation.ts` — Localized navigation helpers
- All existing pages and feature components updated with i18n calls and larger buttons

## Objective

Polish the UI for real-world daily use: calendar-day sprint labels, a global release switcher in the header, bigger action buttons across all tables, and full pt-BR translation. These four improvements touch nearly every component but are each narrow in risk — display changes, sizing, context propagation, and string extraction.

Purpose: Make the app feel production-ready for a Brazilian squad.
Output: All four improvements implemented and verified.
